# Monster-Hunter-UDB
**Monster Hunter Database**

A relational database project built using MySQL to model data from the Monster Hunter game series. 
This database organizes information about games, monsters, weapons, and crafting materials while maintaining proper relationships between entities.

The system is designed using normalized database principles, including one-to-many and many-to-many relationships. 
For example, monsters can appear in multiple games, and weapons can require multiple materials with specific quantities. 
These relationships are handled through junction tables to ensure data integrity and scalability.

**Key features include:**
- Structured storage of Monster Hunter titles and release data
- Monster tracking across multiple games
- Weapon data with attributes such as type, rarity, and attack
- Crafting system with materials and quantities
- Use of foreign keys and relational constraints for consistency
- This project demonstrates core database concepts such as normalization, relational modeling, and SQL querying, and is intended for learning
